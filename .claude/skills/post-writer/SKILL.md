# Post Writer

Write social media posts in Daniel's voice. LinkedIn-first, then adapt for other platforms.

**ALWAYS use this skill when asked to write, draft, or create a social media post, LinkedIn post, tweet, or thread.**

---

## Before you start

Read these files for context (do not skip):
- `context/writing.md` — writing principles, post examples, tone, anti-AI rules
- If available: the Typefully draft scratchpad (original transcript, raw thoughts)

Understand what the user is giving you:
- **Context** = articles, links, research, data. Background material. NOT the user's opinion.
- **Thoughts** = what the user actually thinks, believes, wants to say. THIS is the post.
- Never confuse the two. The post is built from thoughts, informed by context. Context is evidence, not content.

If the input is a task ("write a post about X") rather than a raw thought, you still run Step 1. The task becomes the seed. Step 1 crystallizes it into a postable idea.

## Step 6: Twitter/X adaptation (optional, on request)

This is not "make shorter." This is a re-cut. Same ideas, different architecture.

### Twitter constraints

- 280 characters per tweet. Threads use `---` separator between tweets.
- Default to a single tweet. Only thread when the content genuinely can't compress into one.
- Max 4-5 tweets per thread. Anything longer loses people.
- Tweet 1 must work as a standalone post. If someone never expands the thread, they still get a complete thought.

### Twitter voice (looser than LinkedIn)

- **No capital letters** except for proper names (companies, people, products). Sentence starts are lowercase.
- **No period on the final sentence** of each tweet.
- Slang welcome: "gonna," "kinda," "ngl," "rn" are fine.
- Even more compressed than LinkedIn. Cut every word that isn't pulling weight.
- Fragments are good. Half-sentences are good.
- The tone is someone texting a smart friend, not writing a post.

### Re-cutting process

1. Identify the 1-2 sharpest ideas from the LinkedIn post. Not all of them. Twitter rewards focus.
2. Write tweet 1 as a standalone hook. It should make someone want to read more, or be complete on its own.
3. If threading: each tweet carries exactly one idea. No tweet should need the previous one to make grammatical sense.
4. Apply all Step 5 anti-AI rules. They apply even harder on Twitter because every word is visible.
5. Lowercase pass: downcase everything except proper names.
6. Remove period from the last sentence of each tweet.

### What NOT to do

- Don't just truncate the LinkedIn post. Re-think the structure.
- Don't add hashtags unless the user asks.
- Don't add "thread" or "1/" labels.
- Don't pad tweets to fill 280 chars. Shorter is better.

Output the Twitter version. Wait for user feedback.

## Step 7: Tweet verification (optional, after Step 6)

Read each tweet as if you're scrolling Twitter at 11pm with zero attention span. Be brutal.

### For each tweet, answer:

1. **Would you stop scrolling?** If no, the tweet is boring. Diagnose why.
2. **Is there a sharp edge?** A specific framing, surprising comparison, concrete image, or a take someone could disagree with.
3. **Does it sound like a person or a compressed LinkedIn post?** If you just shrank a paragraph, rewrite from scratch.
4. **First tweet specifically: does it earn the thread?** Someone sees tweet 1 with no context. Does it make them click?

### Common fixes

- **Add stakes or tension.** A prediction is flat. A prediction with a victim or a winner has tension.
- **Use a concrete image early.** Show what it looks like, don't explain the concept.
- **Cut the setup.** If tweet 1 is context and tweet 2 is the interesting part, delete tweet 1.
- **Surprise with the framing.** Same idea, different angle.

Output: pass/fail per tweet with one-line reason. Rewrite failures. Show revised thread.

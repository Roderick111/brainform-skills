# Step 1: Idea Crystallization

The input can be anything: a raw thought, a task ("write about X"), a topic, an article, a conversation transcript, a product you just shipped. Your job is to turn it into one or more atomic, postable ideas.

---

## Process

### 1. Extract ideas

Break the input into atomic ideas. One idea = one post. An idea is atomic when it can stand alone as a complete thought that a reader gets value from without needing the other ideas.

If the input is already atomic, say so and move on.

If the input is a task (e.g., "write a post about our audit tool"), treat it as a seed. Ask yourself: what's the interesting angle? What would make someone stop scrolling? Generate 2-4 possible atomic ideas from this seed.

If the input is a complex thought or article, decompose it. Look for:
- Separate arguments that don't depend on each other
- A buried insight that's stronger than the main point
- An example or story that works better as its own post
- A contrarian take hiding inside a safe take

### 2. Rate each idea

For each idea, assess:

**Strength (1-5):** How interesting is this to someone who isn't you? 1 = anyone could say this. 5 = only someone with your specific experience could say this.

**Readiness:** Is this idea finished or does it need more?
- **Ready** — you have the specifics, the angle, the proof. Can be written now.
- **Needs work** — the core is there but something is missing. Say exactly what: a number, a concrete example, a personal experience, a sharper framing.
- **Seed only** — interesting direction but too vague to write yet. Needs research or more thinking.

**What's missing:** Be specific. "This claim needs a number" or "needs a concrete example of when this happened to you" or "the angle is generic, anyone could say this."

### 3. Prioritize

Rank the ideas. Best first. Explain why in one sentence per idea.

Consider:
- Which idea is most surprising or counterintuitive?
- Which one would you personally stop scrolling for?
- Which one has the most concrete proof behind it?
- Which one is timely (relevant to something happening now)?

### 4. Output format

```
## Ideas extracted

### Idea 1: [one-sentence summary]
Strength: X/5
Readiness: Ready / Needs work / Seed only
Missing: [what's needed, or "nothing" if ready]

### Idea 2: [one-sentence summary]
...

## Priority order
1. Idea X — [why first]
2. Idea Y — [why second]
...

## Recommendation
[Which idea to write now and why. If none are ready, what needs to happen first.]
```

---

## Rules

- Do not write any draft text in this step. Ideas only.
- Do not merge ideas to make one "comprehensive" post. That kills the one-idea rule.
- If the user gives you a clear single idea that's already strong, don't invent extra ideas. Rate it, confirm it's ready, move on.
- Be honest about weak ideas. "This is generic" or "anyone could post this" is useful feedback. Don't rate everything 4/5 to be nice.

Wait for user feedback before proceeding to Step 2.
